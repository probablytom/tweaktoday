Hi!

Thank you so much for checking this project out. It was genuinely a blast to make but it would be even better if people found it fun to use, too!

If you're wanting to contribute, that's incredible. Thanks again. This has been split into two sections: the first is how to contribute if you're not so tech-y, and the second is how to contribute if you can hack on a bit of code, too.

First, though, let's go over the known issues with the site.

Known Issues

  • There is no code of conduct
  • There is no about page
  • Comments appear in the wrong choronological order
  • Photos uploaded from iPhones rotate incorrectly on desktops
  • View code needs to be cleaned up
  • Edit / delete functionality is missing

Non-techy contributions

If you have any bugs or feature requests, let me (Tom) know (you can email tweaktoday -at- tomwallis -dot- net) and he'll add it to the list!

  • Add content to the site! Seriously. Good suggestions for challenges are the bread and butter of these sorts of things. Let's keep them coming!
  • Beginning writing a code of conduct. I'd love to be involved with this if I can. I'd also love to take some direction over how this gets put together from the guide at runyourown.social.
  • An about page needs to be written. It doesn't have to be long, but right now I'm busy with the tech-y side of things. If you can make a start on this I'll be very very grateful!
  • Submit feature requests. This can probably get a lot better, and starting a discussion around things that you as a user want to see is the only way to actually make those improvements. If I don't know about them they can't be implemented. Feedback is a really valuable currency in this game.

Techy contributions

Im so glad you want to contribute to the site! You can find the code in its github repo. The whole thing is django. CSS is a combination of skeleton.css and tufte.css.

  • Fix the comment order!
  • If anybody can figure out how to fix mobile-captured photo rotation when displayed on desktop I'll be forever in your debt. They get spun 90º clockwise. Why?
  • See if you can implement content deletion! Definitely not too hard, if I get around to it I'll implement this, but if you can beat me to it please do!
    If you do attack this, remember how files are managed. If somebody deletes their content the file should also be deleted from the server. Files are stored in the DB as paths to files on disk, so you need to delete that file, then the content.
  • Deleting content is one thing, editing is another. getting editing would be great.
  • Things that need editing / deleting capabilities:
    • Submissions
    • Suggested tasks
    • Comments
    • ...anything else?

Thanks so much!

— Tom.
